# 19.1.3 第三次尝试：数据索引字符串集合

存在一种名为**ASCII**的字符编码格式，该格式为每个字符分配整数值。此处我们发现最大编码值（即所需使用的基数/乘数）为126。我们直接采用该数值——创建与`数据索引英文单词集合`功能相同的结构，但使用`126`作为基数。

```java
public static int asciiToInt(String s) {
    int intRep = 0;
    for (int i = 0; i < s.length(); i += 1) {           
        intRep = intRep * 126;
        intRep = intRep + s.charAt(i);
    }
    return intRep;
}
```

若需支持中文如何处理？最大可能编码值为40959，因此必须将其作为基数。

因此... 要存储一个3个汉字的中文词汇，我们需要容量超过**39万亿**（以T为单位）的数组！这种情况已失控... 让我们探索如何改进这一方案，即采用哈希编码方法。
