#### 异常抛出示例

```java
public void add(T x) {
    if (x == null) {
        throw new IllegalArgumentException("禁止添加 null");
    }
    // ...后续正常逻辑
}
```

---

#### 异常处理优势

1. **控制程序流程**  
    主动决定错误发生时的终止点
    
2. **提升调试效率**  
    ▸ 明确异常类型（如`IllegalArgumentException`）  
    ▸ 自定义错误信息（"禁止添加 null"）
    

---

#### 替代处理方案对比

|方法|实现方式|适用场景|
|---|---|---|
|**前置校验抛异常**|在方法入口检查参数合法性|关键参数校验|
|**静默处理**|跳过非法操作不报错|非致命性错误|
|**空值兼容**|修改逻辑支持 null 值处理|需要兼容 null 的场景|

---

#### 关键要点

▸ 使用 `throw` 主动抛出异常对象（如 `new IllegalArgumentException()`）  
▸ 必须用 Javadoc 明确方法约束（如 `@throws` 声明）  
▸ 异常类型选择原则：

- `IllegalArgumentException` → 参数不合法
- `NullPointerException` → 意外空值访问
- `IllegalStateException` → 对象状态异常
