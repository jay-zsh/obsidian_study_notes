
# 23.3 总结  
---

## 图遍历概述  
- 树结构中的遍历方法可推广至图结构。给定源顶点时，可通过以下方式访问顶点：  
    - **DFS 前序**：DFS 在各顶点上调用的顺序  
    - **DFS 后序**：DFS 从各顶点返回的顺序  
    - **BFS**：距离源节点的层级顺序（在树中即层级遍历）  

---

## BFS  
- 与 DFS 不同，BFS 天然采用迭代（非递归）解法：先访问源顶点 `s`，再访问距离 `s` 为 1 的顶点，接着距离为 2 的顶点，依此类推。  
- BFS 使用待探索顶点的**边缘集**（fringe），该集合以队列实现：新顶点从队尾入列，待访问顶点从队首出列。  
- BFS 可解决最短路径问题（最小化源点到其他顶点的边数）。若需重建路径，需在遍历中记录各顶点的 `edgeTo`。  

---

## 图的实现  
- 图 API 的选择决定客户端代码的编写方式：不同 API 对特定任务的简化程度及对运行时/内存的影响各异。  
- 实现方案包括：  
    - **邻接矩阵**：二维布尔数组，标记顶点对是否相邻  
    - **边列表**：存储图中所有边的集合  
    - **邻接表**：通过顶点编号索引的列表数组，每个索引存储与该顶点相连的所有顶点（最常用方案）
