# 13.10 练习
## 基础题
分析以下代码的运行时间复杂度（以N为参数）：

```java
for (int i = 0; i < N; i++) {
    int j = 0;
    while (j < N) {
        j = N;
    }
}
```


给定$f(N) = 2N$，判断以下陈述真伪：

☐ $f(N) \in \Theta(1)$  
☑ $f(N) \in \Theta(N)$  
☐ $f(N) \in \Theta(N^2)$  
☐ $f(N) \in O(1)$  
☑ $f(N) \in O(N)$  
☑ $f(N) \in O(N^2)$  

**关键分析**：内层循环首次执行即设置j=N，因此仅运行一次。整体运行时间由外层循环主导，时间复杂度为线性。

## 应用题
判断以下陈述真伪：

1. 若$f(N) \in \Theta(N^2)$，输入规模从N增至2N时耗时约为4倍  
☑ 正确（符合渐近行为定义）

2. 若$f(N) \in \Theta(N^2)$，输入规模从100增至200时耗时约为4倍  
☑ 错误（小规模输入可能未进入渐近区间）

## 元认知题
为何使用渐近分析而非实际计时（如Lab 3的秒表工具）？

- **硬件无关性**：不同计算机的运算速度存在显著差异（受架构/硬件/环境温度等影响）
- **规模可行性**：无法在超大规模输入下进行实际测试
- **语言通用性**：相同算法在不同语言中实际耗时不同（如C比Python快10-400倍），但渐近复杂度一致
- **边界情况**：最坏情况可能难以通过实验复现
