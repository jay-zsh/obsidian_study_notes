# 39.2 最优压缩与柯尔莫哥洛夫复杂性

---

## 柯尔莫哥洛夫复杂性

我们将比特流 `B` 的**柯尔莫哥洛夫复杂性**定义为输出 `B` 的最短比特流 CBCB​。令 _Java-柯尔莫哥洛夫复杂性_ $KJ(B)$ 为生成 `B` 的最短 Java 程序。

注意，对于任何比特流 $B$，$K(B)$ 一定存在。然而，找到并证明 $K(B)$ 可能很困难甚至不可能。

---

### 语言与复杂性

需要注意的一个重要点是，柯尔莫哥洛夫复杂性是语言无关的。要在另一种语言中运行任何程序，我只需编写一个解释器。例如，如果我想运行一个不易翻译为 Java 的 Python 程序，我可以编写一个 Java 解释器来读取 Python 程序的文本并运行它。在这种情况下，$KJ(B) \leq KP(B) + I$，其中 $I$ 是解释器的长度（一个常数值）。

这突显了关于柯尔莫哥洛夫复杂性的一个深刻事实：无论我们选择哪种语言作为压缩算法，大多数比特流本质上是不可压缩的。

考虑一个 1,000,000 位的比特流。在所有可能的压缩算法中，只有 1/24999999 的比特流有机会被压缩超过 50%（即 499,999 位或更少）。

---

### 不可计算性

关于柯尔莫哥洛夫复杂性的另一个重要事实是它是不可计算的。此事实的证明在此提供。

实际上，这意味着无法编写一个"完美"（最优）的压缩算法，因为我们甚至无法计算最短程序的长度！
