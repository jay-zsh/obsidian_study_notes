
# 11.2 子类型多态与显式高阶函数对比

我们已了解继承机制如何通过重写超类方法或在子类中编写全新方法，实现基于超类的代码复用与局部修改。利用_多态_特性，继承还能帮助我们设计通用数据结构和算法。

多态本质即"多种形态"。在Java中，多态指对象可具备多种形态或类型。面向对象编程中，一个对象既可视作自身类的实例，也可视为其超类、超类的超类等多级继承体系中的实例。

以静态类型为Deque的变量`deque`为例。执行`deque.addFirst()`方法时，具体调用的方法实现将根据运行时的动态类型决定。如前一章所述，Java通过动态方法选择机制确定具体调用的方法。

假设我们需要编写Python程序输出两个对象中较大值的字符串表示。现有两种实现方案：

---

## 1. 显式高阶函数方案

```python
def print_larger(x, y, compare, stringify):
    if compare(x, y):
        return stringify(x)
    return stringify(y)
```

---

## 2. 子类型多态方案

```python
def print_larger(x, y):
    if x.largerThan(y):
        return x.str()
    return y.str()
```

显式高阶函数方案通过通用比较逻辑实现对象大小判断。而子类型多态方案则将决策权赋予对象本身——实际调用的`largerThan`方法完全取决于x和y的具体类型。
